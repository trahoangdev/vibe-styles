# Contributing to Vibe Styles

Thank you for your interest in contributing to Vibe Styles! We welcome contributions from the community to help make this design system explorer even better.

## How Can I Contribute?

### Reporting Bugs

- Search existing issues to see if the bug has already been reported.
- If not, open a new issue with a clear title and description, including steps to reproduce the bug.
- Include information about your environment (browser, OS, etc.).

### Suggesting Enhancements

- Open a new issue with a detailed description of the proposed enhancement.
- Explain why this enhancement would be useful to other users.

### Pull Requests

1. **Fork the repository** to your own GitHub account.
2. **Clone your fork** to your local machine:
   ```sh
   git clone https://github.com/trahoangdev/vibe-styles.git
   ```
3. **Create a new branch** for your changes:
   ```sh
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** and ensure they follow the project's coding style.
5. **Commit your changes** with a clear and descriptive commit message.
6. **Push your branch** to your fork:
   ```sh
   git push origin feature/your-feature-name
   ```
7. **Open a Pull Request** against the `main` branch of the original repository.

## Development Setup

1. Install dependencies:
   ```sh
   npm install
   ```
2. Start the development server:
   ```sh
   npm run dev
   ```
3. Run linting:
   ```sh
   npm run lint
   ```

## Design Principles

- **Premium Aesthetics**: All contributions should maintain a high-end, modern look and feel.
- **Performance**: Ensure animations and transitions stay smooth (60fps).
- **Usability**: Keep the interface intuitive and easy to navigate.

## Code of Conduct

Please be respectful and professional in all your interactions with the community.
